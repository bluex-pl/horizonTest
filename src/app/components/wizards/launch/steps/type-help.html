Ephemeral instances

Instance data is lost when the instance is deleted.

Boot source
Use boot from image or boot from snapshot.

Prerequisites
Have following components created before launching the instance:

  • Image or snapshot
  • Security group
  • Key pair – to log into the instance
  • Network – if you want to connect using an External network, you need to allocate and assign a Floating IP

Persistent instances

Instance data is saved after the instance is deleted. New instances can use an existing volume. You can also restore a deleted instance from a volume.

Boot source
Use volume, volume snapshot (creates a new volume) or image (creates new volume).

Prerequisites
Have following components created before launching the instance:

  • Volume (if using boot from volume option)
    If you choose boot from image or boot from image snapshot then you don't need an existing volume. A new volume is created during the launch instance process.
  • Security group
  • Key pair – to log into the instance
  • Network – if you want to connect using an External network, you need to allocate and assign a Floating IP
